Data preparation

The data can be prepared for different software tools as follows:

SARvey

Use the sarvey_export script to export the time series data to a shapefile or geopackage file. For example:

$ sarvey_export outputs/p2_coh80_ts.h5 -o outputs/shp/p2_coh80_ts.shp

MintPy

Use the save_qgis script to export the time series data to a shapefile. For example:

$ mintpy save_qgis timeseries_ERA5_ramp_demErr.h5 -g inputs/geometrygeo.h5

MiaplPy

Use the save_qgis script to export the time series data to a shapefile. For example:

$ mintpy save_qgis timeseries_ERA5_ramp_demErr.h5 -g inputs/geometrygeo.h5

GMTSAR

GMTSAR provides the time series as raster in grd format. It is possible to convert the time series data to vector and handle them in InSAR Explorer. Script to convert to shapefile/geopackage to be added

Alternatively, you can directly use the raster grd files created by GMTSAR in InSAR Explorer. The directory structure should be as follows:

-| GMTSAR_output/
  -| vel_*.grd
  -| YYYYMMDD_*.grd
  -| YYYYMMDD_*.grd
  -| ...

Open the original vel_*.grd file created by GMTSAR in QGIS. The grd files associated with the time series should be located in the same directory as the vel_*.grd file. InSAR Explorer will automatically detect the time series files
